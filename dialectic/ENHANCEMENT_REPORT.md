### 5. N+1 Query in Message Ancestry (Performance)
**File**: `api/main.py:597-599`

```python
# CURRENT (RECURSIVE PYTHON CALLS)
from operations import get_thread_messages
all_messages = await get_thread_messages(db, thread_id, include_ancestry=True)
```

The `get_thread_messages` function recursively fetches parent threads one by one.

**Fix**: Replace with recursive CTE:
```sql
WITH RECURSIVE ancestry AS (
    -- Base: current thread
    SELECT id, parent_thread_id, fork_point_message_id
    FROM threads WHERE id = $1

    UNION ALL

    -- Recursive: parent threads
    SELECT t.id, t.parent_thread_id, t.fork_point_message_id
    FROM threads t
    JOIN ancestry a ON t.id = a.parent_thread_id
)
SELECT m.* FROM messages m
JOIN ancestry a ON m.thread_id = a.id
WHERE NOT m.is_deleted
  AND (
      a.id = $1  -- Current thread: all messages
      OR m.sequence <= (
          SELECT fork_point_sequence FROM threads WHERE id = a.id
      )
  )
ORDER BY m.created_at, m.sequence
```

### 7. WebSocket Reconnection Race Conditions (Frontend)
**File**: `frontend/index.html` - reconnection logic

Multiple race conditions identified:
1. Messages sent during reconnection are lost
2. Rapid disconnect/reconnect can create duplicate connections
3. No message queue for offline operation

**Fix**: Add message queue and connection state machine:
```javascript
// Connection state machine
const ConnectionState = {
    DISCONNECTED: 'disconnected',
    CONNECTING: 'connecting',
    CONNECTED: 'connected',
    RECONNECTING: 'reconnecting'
};

// Message queue
const messageQueue = [];

function sendMessage(msg) {
    if (state.connectionState !== ConnectionState.CONNECTED) {
        messageQueue.push(msg);
        return;
    }
    state.ws.send(JSON.stringify(msg));
}

function onConnected() {
    state.connectionState = ConnectionState.CONNECTED;
    // Flush queue
    while (messageQueue.length > 0) {
        const msg = messageQueue.shift();
        state.ws.send(JSON.stringify(msg));
    }
}
```

### 8. Exponential Backoff Missing (Frontend)
**File**: `frontend/index.html` - `scheduleReconnect()`

Fixed 3-second reconnect interval can overwhelm server during outages.

**Fix**:
```javascript
const RECONNECT_BASE_MS = 1000;
const RECONNECT_MAX_MS = 30000;

function scheduleReconnect() {
    const delay = Math.min(
        RECONNECT_BASE_MS * Math.pow(2, state.reconnectAttempts),
        RECONNECT_MAX_MS
    ) + Math.random() * 1000;  // Jitter

    state.reconnectTimeout = setTimeout(() => {
        state.reconnectAttempts++;
        connect();
    }, delay);
}

// Reset on successful connection
function onConnected() {
    state.reconnectAttempts = 0;
    ...
}
```

### 17. Agent-Native Gaps (Architecture)

- Fork threads
