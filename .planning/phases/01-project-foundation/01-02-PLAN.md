  <files>
    mobile/eas.json

  </files>
  <action>
    1. Install EAS CLI globally if needed and initialize EAS in the project:
       ```bash
       cd /root/DwoodAmo/mobile
       npx eas-cli@latest build:configure
       ```

       This will:
       - Prompt to log in to Expo account (or create one)
       - Create eas.json with default configuration
       - Add EAS project ID to app.config.js

       If running non-interactively or no account exists, create eas.json manually.

    2. Update mobile/eas.json with three build profiles:
       ```json
       {
         "cli": {
           "version": ">= 13.0.0"
         },
         "build": {
           "development": {
             "developmentClient": true,
             "distribution": "internal",
             "ios": {
               "simulator": true
             },
             "android": {
               "buildType": "apk"
             }
           },
           "preview": {
             "distribution": "internal",
             "android": {
               "buildType": "apk"
             }
           },
           "production": {
             "autoIncrement": true
           }
         },
         "submit": {
           "production": {}
         }
       }
       ```

       Profile purposes:
       - development: Dev builds for simulator/emulator testing
       - preview: Internal testing builds (APK for Android, Ad Hoc for iOS)
       - production: Store-ready builds with auto-incrementing version

    3. Ensure mobile/app.config.js has the extra.eas.projectId field (added by eas build:configure).
       If configuring manually, add a placeholder comment:
       ```javascript
       extra: {
         eas: {
           projectId: 'your-project-id', // Set by running: npx eas-cli build:configure
         },
       },
       ```

    4. Verify eas.json is valid:
       ```bash
       npx eas-cli@latest build:list --non-interactive || echo "EAS not fully configured - OK for now"
       ```

    Note: Full EAS functionality requires EXPO_TOKEN. The configuration is valid even without it.

<task type="auto">
  <name>Task 2: Create GitHub Actions CI workflow</name>
  <files>
    .github/workflows/mobile-ci.yml
  </files>
  <action>
    1. Create the workflow directory if it doesn't exist:
       ```bash
       mkdir -p /root/DwoodAmo/.github/workflows
       ```

    2. Create .github/workflows/mobile-ci.yml:
       ```yaml
       name: Mobile CI

       on:
         pull_request:
           branches: [main, master]
           paths:
             - 'mobile/**'
             - '.github/workflows/mobile-ci.yml'
         push:
           branches: [main, master]
           paths:
             - 'mobile/**'
             - '.github/workflows/mobile-ci.yml'

       jobs:
         lint-and-test:
           name: Lint, Type Check, Test
           runs-on: ubuntu-latest
           defaults:
             run:
               working-directory: mobile
           steps:
             - uses: actions/checkout@v4

             - uses: actions/setup-node@v4
               with:
                 node-version: 22
                 cache: npm
                 cache-dependency-path: mobile/package-lock.json

             - name: Install dependencies
               run: npm ci

             - name: Lint
               run: npx expo lint

             - name: Type check
               run: npx tsc --noEmit

             - name: Test
               run: npm test -- --ci --coverage

             - name: Upload coverage
               uses: actions/upload-artifact@v4
               if: always()
               with:
                 name: coverage-report
                 path: mobile/coverage/
                 retention-days: 7

         build:
           name: EAS Build
           needs: lint-and-test
           if: github.ref == 'refs/heads/main' || github.ref == 'refs/heads/master'
           runs-on: ubuntu-latest
           defaults:
             run:
               working-directory: mobile
           steps:
             - uses: actions/checkout@v4

             - uses: actions/setup-node@v4
               with:
                 node-version: 22
                 cache: npm
                 cache-dependency-path: mobile/package-lock.json

             - name: Setup Expo and EAS
               uses: expo/expo-github-action@v8
               with:
                 eas-version: latest
                 token: ${{ secrets.EXPO_TOKEN }}

             - name: Install dependencies
               run: npm ci

             - name: Build on EAS
               run: eas build --platform all --non-interactive --no-wait
       ```

       Key design decisions:
       - Uses paths filter to only run on mobile/ changes
       - lint-and-test runs on all PRs (fast feedback)
       - build job only runs on main branch (expensive EAS builds)
       - --no-wait means CI doesn't block waiting for EAS build completion
       - Coverage uploaded as artifact for review

    3. Verify YAML is valid:
       ```bash
       python3 -c "import yaml; yaml.safe_load(open('/root/DwoodAmo/.github/workflows/mobile-ci.yml'))"
       ```
  </action>

    - Build job has condition for main/master branch only
  </verify>
