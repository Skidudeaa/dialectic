  <name>Task 1: Add search vector column and GIN index</name>
  <files>dialectic/schema.sql</files>
  <action>
Add full-text search infrastructure to the messages table:

1. Add search_vector column to messages table:
   ```sql
   ALTER TABLE messages ADD COLUMN IF NOT EXISTS search_vector tsvector;
   ```

2. Create GIN index for fast search:
   ```sql
   CREATE INDEX IF NOT EXISTS idx_messages_search
   ON messages USING GIN (search_vector);
   ```

3. Create trigger to auto-update search vector on insert/update:
   ```sql
   CREATE OR REPLACE FUNCTION messages_search_trigger() RETURNS trigger AS $$
   BEGIN
     NEW.search_vector := to_tsvector('english', COALESCE(NEW.content, ''));
     RETURN NEW;
   END
   $$ LANGUAGE plpgsql;

   CREATE TRIGGER messages_search_update
     BEFORE INSERT OR UPDATE ON messages
     FOR EACH ROW EXECUTE FUNCTION messages_search_trigger();
   ```

4. Add created_at index for date range filtering:
   ```sql
   CREATE INDEX IF NOT EXISTS idx_messages_created_at
   ON messages (thread_id, created_at DESC);
   ```

5. Add backfill statement for existing messages:
   ```sql
   UPDATE messages SET search_vector = to_tsvector('english', COALESCE(content, ''))
   WHERE search_vector IS NULL;
   ```

Place these additions in a new section "FULL-TEXT SEARCH" after the REAL-TIME section.

Add two new endpoints to FastAPI:

1. **GET /messages/search** - Full-text search endpoint:
   - Query params: q (required), thread_id (optional), date_from (optional), date_to (optional), speaker_type (optional), limit (default 50)
   - Uses plainto_tsquery for search
   - Returns ts_headline for snippet with highlighted matches
   - Returns ts_rank for relevance scoring
   - Joins with room_memberships to ensure user can only search rooms they belong to
   - Orders by rank DESC, created_at DESC

   Response model:
   ```python
   class SearchResultResponse(BaseModel):
       id: UUID
       thread_id: UUID
       content: str
       snippet: str  # With highlighted matches
       sender_name: str
       speaker_type: str
       created_at: datetime
       rank: float
   ```

2. **GET /threads/{thread_id}/messages/context** - Context around a message:
   - Query params: message_id (required), context (default 25)
   - Gets the target message's sequence number
   - Returns messages where sequence BETWEEN (target - context) AND (target + context)
   - Verifies room membership via thread -> room -> room_memberships

   Uses existing MessageResponse model.

Both endpoints require authentication via the token query parameter (existing pattern in codebase).
