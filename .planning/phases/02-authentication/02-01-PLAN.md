  <name>Task 1: Database Schema for Authentication</name>
  <files>dialectic/schema.sql</files>
  <action>
Add authentication tables to the existing schema.sql file. Append after the existing memory_versions table:

1. **user_credentials** table:
   - user_id UUID PRIMARY KEY REFERENCES users(id)
   - email TEXT UNIQUE NOT NULL
   - email_verified BOOLEAN NOT NULL DEFAULT FALSE
   - password_hash TEXT NOT NULL
   - created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
   - updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
   - Index on email

2. **verification_codes** table:
   - id UUID PRIMARY KEY DEFAULT gen_random_uuid()
   - user_id UUID NOT NULL REFERENCES users(id)
   - code TEXT NOT NULL (6 digits)
   - purpose TEXT NOT NULL ('email_verification' | 'password_reset')
   - created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
   - expires_at TIMESTAMPTZ NOT NULL
   - used_at TIMESTAMPTZ (null until used)
   - Index on user_id

3. **user_sessions** table:
   - id UUID PRIMARY KEY DEFAULT gen_random_uuid()
   - user_id UUID NOT NULL REFERENCES users(id)
   - refresh_token_hash TEXT NOT NULL
   - device_info JSONB (optional metadata)
   - created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
   - last_used_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
   - expires_at TIMESTAMPTZ NOT NULL
   - revoked_at TIMESTAMPTZ (null until revoked)
   - Index on user_id, index on refresh_token_hash

4. **user_pins** table (for biometric fallback):
   - user_id UUID PRIMARY KEY REFERENCES users(id)
   - pin_hash TEXT NOT NULL
   - created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
   - updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()

Add appropriate indexes for all lookup patterns. Use gen_random_uuid() for auto-generated UUIDs.

  <action>
Create the auth module at dialectic/api/auth/:

**1. requirements.txt** - Add to existing:
```
pyjwt>=2.9.0
pwdlib[argon2]>=0.2.0
pyotp>=2.9.0
```

**2. utils.py** - JWT and password utilities:
- SECRET_KEY from env var JWT_SECRET_KEY (required, no default)
- ALGORITHM = "HS256"
- ACCESS_TOKEN_EXPIRE_MINUTES = 15
- REFRESH_TOKEN_EXPIRE_DAYS = 90
- password_hash = PasswordHash.recommended() from pwdlib
- verify_password(plain, hashed) -> bool
- get_password_hash(password) -> str
- create_access_token(data: dict, expires_delta: Optional[timedelta]) -> str
- create_refresh_token(data: dict) -> str
- decode_token(token: str) -> dict (raises jwt.InvalidTokenError)
- generate_verification_code() -> str (6 random digits using secrets.choice)

**3. schemas.py** - Pydantic models:
- SignUpRequest: email (EmailStr), password (min 8 chars), display_name (str)
- SignInRequest: email (EmailStr), password (str)
- TokenResponse: access_token, refresh_token, token_type="bearer", user_id (UUID)
- RefreshRequest: refresh_token (str)
- VerifyEmailRequest: code (str, 6 digits)
- ForgotPasswordRequest: email (EmailStr)
- ResetPasswordRequest: email (EmailStr), code (str), new_password (min 8 chars)
- UserResponse: id (UUID), email, display_name, email_verified

**4. dependencies.py** - Auth dependencies:
- get_current_user(token: str = Depends(oauth2_scheme)) -> User
  - Decode JWT, verify type="access", fetch user from DB
  - Raise HTTPException 401 if invalid
- get_current_verified_user(user = Depends(get_current_user)) -> User
  - Also requires email_verified=True
  - Raise HTTPException 403 if not verified

**5. routes.py** - Auth endpoints:
- POST /auth/signup - Create user_credentials + users record, generate verification code, return tokens
  - Check email not already taken (409 if exists)
  - Hash password with Argon2
  - Create user record in users table first
  - Create user_credentials record
  - Generate 6-digit verification code, store in verification_codes (30 min expiry)
  - Create session, return tokens
  - Note: Email sending is out of scope - log the code for now

- POST /auth/login - Verify credentials, return tokens
  - Find user by email, verify password
  - Return 401 if invalid credentials
  - Create new session, return tokens
  - Update last_used_at on session

- POST /auth/refresh - Exchange refresh token for new access token
  - Verify refresh token JWT, check session not revoked
  - Generate new access token (same refresh token)
  - Update session last_used_at
  - Return 401 if invalid/expired/revoked

- POST /auth/logout - Revoke session
  - Mark session as revoked (set revoked_at)
  - Return success

- POST /auth/verify-email - Verify email with 6-digit code
  - Find valid, unused code for user
  - Mark email_verified=True on user_credentials
  - Mark code as used
  - Return 400 if invalid/expired/used

- POST /auth/forgot-password - Request password reset
  - Find user by email
  - Generate 6-digit code, store with purpose='password_reset' (30 min expiry)
  - Always return 200 (don't leak email existence) - wait, CONTEXT.md says explicit error
  - Return 404 if no account exists for email

- POST /auth/reset-password - Reset password with code
  - Verify code is valid and unused
  - Update password_hash
  - Mark code as used
  - Revoke all existing sessions for user
  - Auto-login: create new session, return tokens

**6. main.py** - Include auth router:
- Add: `from api.auth.routes import router as auth_router`
- Add: `app.include_router(auth_router, prefix="/auth", tags=["auth"])`

Multi-device limit: When creating a session, count user's active sessions. If >= 5, revoke the oldest one (by last_used_at).

Error handling: Use HTTPException with appropriate status codes (400, 401, 403, 404, 409).
