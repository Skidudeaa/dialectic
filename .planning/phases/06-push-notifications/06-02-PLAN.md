<task type="auto">
  <name>Task 3: Token registration with backend API</name>
  <files>mobile/services/notifications/registration.ts</files>
  <action>
Create mobile/services/notifications/registration.ts:

1. registerForPushNotifications(userId: string): Promise<string | null>
- Import notificationService from ./index
- Call notificationService.requestPermission()
- If not granted, return null
- Call notificationService.getExpoPushToken()
- If no token, return null
- POST to /notifications/tokens with:
  - expo_push_token: token
  - platform: Platform.OS
  - device_name: Device.deviceName
- Return the token on success, null on error (log error)
- Use api client from @/services/api

2. unregisterPushToken(token: string): Promise<void>
- DELETE to /notifications/tokens with expo_push_token in body
- Log success/error but don't throw

3. refreshPushToken(): Promise<string | null>
- Re-register token (useful for token refresh events)
- Simply calls registerForPushNotifications with current user ID

Follow patterns from existing services (api.ts import, error handling).
  </action>
  <verify>
Run: `cd mobile && npx tsc --noEmit 2>&1 | grep registration` should show no errors.
Import check: `cd mobile && node -e "require('./services/notifications/registration')" 2>&1 || echo "TS not compiled, check tsc"`
  </verify>
  <done>Token registration functions ready to register/unregister push tokens with backend.</done>
