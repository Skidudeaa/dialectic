<task id="1" type="auto">
  <name>Add thread genealogy endpoint</name>
  <files>dialectic/api/main.py</files>
  <action>
Add `GET /rooms/{room_id}/genealogy` endpoint that returns the full thread tree for a room.

Use PostgreSQL recursive CTE to efficiently fetch all threads with depth tracking:

```python
class ThreadNodeResponse(BaseModel):
    id: UUID
    parent_thread_id: Optional[UUID]
    fork_point_message_id: Optional[UUID]
    title: Optional[str]
    message_count: int
    created_at: datetime
    depth: int
    children: List['ThreadNodeResponse'] = []

ThreadNodeResponse.model_rebuild()  # For self-reference

@app.get("/rooms/{room_id}/genealogy")
async def get_thread_genealogy(
    room_id: UUID,
    token: str = Query(...),
    max_depth: int = Query(20, ge=1, le=50),
    db=Depends(get_db),
):
```

The recursive CTE should:
1. Start with root threads (parent_thread_id IS NULL)
2. Recursively join child threads
3. Track depth level (0-based)
4. Include message count per thread
5. Limit recursion to max_depth (default 20) to prevent runaway queries

Build tree structure in Python after fetching flat list:
- Create dict mapping id -> ThreadNodeResponse
- Iterate and assign children to parents
- Return roots (threads with no parent)

Order by depth ASC, then created_at ASC for consistent ordering.

Run server and test:
```bash
curl "http://localhost:8000/rooms/{room_id}/genealogy?token={token}" | jq
```
Verify response is JSON array of root threads with nested children.
  </verify>
  <done>Genealogy endpoint returns tree structure with all threads, depths, and message counts</done>
</task>
